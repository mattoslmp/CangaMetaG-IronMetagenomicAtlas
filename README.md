# CangaMetaG Iron Metagenomic Atlas

Interactive Streamlit atlas and complete figure-reproduction repository for the manuscript:

**An iron metagenomic atlas reveals biogeochemical cycling potential in Amazonian lateritic lake sediments**

**Authors:** Leandro de Mattos Pereira; José Augusto Pires Bittencourt; Vitor Cirilo Araujo Santos; Ronnie Alves; Eder Pires; Prafulla Kumar Sahoo; José Tasso Felix Guimarães; Bruno Garcia Simões; Renato R. Moreira-Oliveira; Guilherme Oliveira; Gisele Lopes Nunes  
**Target journal:** *The ISME Journal*  
**Manuscript date:** 19 July 2026

## Scope

This repository contains the unchanged original application interface, the processed data used by the app and article, all main and supplementary figures, Supplementary Tables 1–12, and the scripts required to reproduce the analyses and figures. The final revision preserves the scientific values and interpretations while improving figure legibility, correcting Figure 8 comparison labels, filtering the displayed KEGG/KEMET module calls, and documenting the external iron-rich environments.

## Final RDA model

The default interactive RDA uses the same internal processing as the article:

1. read `data/resultado.cds.otu.tab`, `data/resultado.cds.tax.tab`, and `data/fiqui2.xlsx`;
2. pool dry and rainy counts by the ten physicochemical sampling positions;
3. aggregate at genus level and retain the 80 most abundant genera;
4. apply a Hellinger transformation;
5. average physicochemical replicates by `SampleMM`;
6. standardize the retained predictors;
7. fit the constrained model and evaluate it with 999 permutations using seed 42.

The default article predictors are:

```text
Fe2O3, SiO2, Al2O3, TOT/S, Cu, Pb
```

Expected article-wide result:

```text
n = 10 sampling positions
constrained R2 = 0.6605827831733051
pseudo-F = 0.9731132518103761
permutation P = 0.534
```

## Figure 8 correction

Figure 8 is generated by `scripts/regenerate_figure8_comparison_labels_v9.py`. The script reads the dry- and rainy-season worksheets in Supplementary Table 5 and prints the exact comparison beside each KO/gene marker. Positive values indicate enrichment in the first named group; negative values indicate enrichment in the second named group.

## KEGG/KEMET module display filter

Supplementary Figures 37, 38, 40, 41, and 67 are generated by `scripts/regenerate_module_figures_filtered_v9.py`.

Only module calls classified as:

- `Complete`; or
- `1 block missing`

are drawn. Calls with more than one missing block are blank and are not displayed. A row is retained only when at least one sample has a displayed call. The complete three-state source matrices remain available under `data/final_kegg_st8_update/` for auditability.

## Installation

### Conda

```bash
git clone https://github.com/mattoslmp/CangaMetaG-IronMetagenomicAtlas.git
cd CangaMetaG-IronMetagenomicAtlas
conda env create -f environment.yml
conda activate cangametag-iron-atlas
```

### Python virtual environment

```bash
python3 -m venv .venv
source .venv/bin/activate
python -m pip install --upgrade pip
python -m pip install -r requirements.txt
```

Additional system tools used for validation and document rendering:

```bash
sudo apt update
sudo apt install -y libreoffice imagemagick zstd git git-lfs rsync
```

## Run the app

```bash
streamlit run app.py --server.port 8502 --server.address 0.0.0.0
```

Portable launcher:

```bash
bash run_app_no_root.sh --server.port 8502 --server.address 0.0.0.0 --server.headless true
```

The original page structure, CSS, navigation, modules, colours, cards, forms, maps, tables, and non-RDA functionality are preserved.

## Reproduce every figure

```bash
bash scripts/run_all_figures.sh
```

The detailed figure-by-figure instructions are in:

```text
FIGURE_REPRODUCTION_GUIDE.md
Final_Figures_and_Scripts/final_figure_script_manifest.csv
scripts/README.md
```

## Validate the final package

```bash
python scripts/validate_all_figures_v9.py
```

The validator checks:

- the 8 main figures and 69 supplementary figures in PNG, PDF, and SVG;
- minimum raster dimensions and file integrity;
- exact Figure 8 comparison labels against Supplementary Table 5;
- module-filter implementation and filtered source matrices;
- Supplementary Table 12;
- image counts, captions, and table references in the main and supplementary DOCX files;
- equality of article and app figure copies;
- the complete figure-script-data manifest.

## Supplementary Table 12

`tables/Supplementary_Table_12_external_iron_rich_environment_descriptions.xlsx` documents every external comparison environment. It includes location, available coordinates, environment and sample type, data layer, project/database origin, IMG/GOLD/NCBI/SRA accessions, source links, and ecological notes. Fields absent from the packaged metadata are explicitly marked `Not reported`; no missing values were invented.
