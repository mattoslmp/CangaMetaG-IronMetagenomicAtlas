Reproducibility repository and interactive Streamlit atlas for the manuscript:

**Iron-rich Amazonian lateritic lake sediments harbor diverse microbial communities with biogeochemical potential relevant to carbon and methane cycling**

**Authors:** Leandro de Mattos Pereira; José Augusto Pires Bittencourt; Vitor Cirilo Araujo Santos; Ronnie Alves; Eder Pires; Prafulla Kumar Sahoo; José Tasso Felix Guimarães; Bruno Garcia Simões; Renato R. Moreira-Oliveira; Guilherme Oliveira; Gisele Lopes Nunes  
**Affiliation:** Instituto Tecnológico Vale, Belém, PA, Brazil  
**Corresponding authors:** Gisele Lopes Nunes and Leandro de Mattos Pereira  
**Target journal:** *The ISME Journal*  
**Article type:** Original Article  
**Manuscript date:** 19 July 2026

## Scope

The repository contains the Streamlit application, independent analysis and figure-generation scripts, processed source data, supplementary tables, publication-figure provenance, validation records and figure-to-script manifests associated with the Amazonian lateritic-lake sediment metagenomic atlas.

The app covers taxonomic profiles, alpha and beta diversity, NMDS, redundancy analysis, KO biomarkers, differential abundance, KEGG/KEMET module completeness, FeGenie summaries, MAG quality and taxonomy, antiSMASH biosynthetic gene clusters, ST8 comparisons and environmental integration.

## Canonical RDA model

The exact article RDA implementation is preserved in `scripts/consolidate_final_publication.py`. Domain-separated RDA panels in Main Figures 4 and 5 are generated by `scripts/generate_final_domain_taxonomy_figures.py`. The original R workflow is preserved in `scripts/RDA_Taxon_env_Gi.R`.

The workflow pools dry- and rainy-season CDS counts by ten sampling positions, aggregates the community at genus level, retains the 80 genera with the largest total counts, applies a Hellinger transformation, averages physicochemical replicates by `SampleMM`, standardizes environmental variables and evaluates the constrained model with 999 permutations.

Predictors are evaluated in this fixed order:

```text
Fe2O3, SiO2, Al2O3, TOT/C, TOT/S, LOI, V, Cu, Pb, Zr, Ce
```

Near-zero-variance predictors are removed, pairwise absolute correlation must remain below 0.92, and selection stops after six predictors. The final manuscript model is:

```text
Fe2O3, SiO2, Al2O3, TOT/S, Cu, Pb
```

Cu and Pb are the trace-metal variables retained in the model. The default interactive RDA uses the same sample mapping, aggregation, transformation, predictor order, collinearity rule, constrained fit, seed and permutation test as the article script.

Expected article-wide result:

```text
n positions = 10
constrained R2 = 0.6605827831733045
pseudo-F = 0.9731132518103737
permutation P = 0.534
```

## Installation

### Conda

```bash
git clone https://github.com/mattoslmp/CangaMetaG-IronMetagenomicAtlas.git
cd CangaMetaG-IronMetagenomicAtlas
conda env create -f environment.yml
conda activate cangametag-iron-atlas
```

### Python virtual environment

```bash
git clone https://github.com/mattoslmp/CangaMetaG-IronMetagenomicAtlas.git
cd CangaMetaG-IronMetagenomicAtlas
python3 -m venv .venv
source .venv/bin/activate
python -m pip install --upgrade pip
python -m pip install -r requirements.txt
```

## Run the app

```bash
streamlit run app.py --server.port 8502 --server.address 0.0.0.0
```

Portable no-root launch:

```bash
bash run_app_no_root.sh --server.port 8502 --server.address 0.0.0.0 --server.headless true
```

Optional administrator credentials must be supplied only through environment variables:

```bash
export CANGAMETAG_ADMIN_USER='admin'
export CANGAMETAG_ADMIN_PASSWORD='replace-with-a-strong-password'
```

Never commit passwords, API tokens, `.env` files or `.streamlit/secrets.toml`.

## Validate the article RDA

```bash
python scripts/validate_article_rda_consistency.py
```

The validator checks the default model, expected article statistics and SHA-256 hashes of the RDA-containing publication files.

## Reproduce the RDA and taxonomic figures

```bash
python scripts/consolidate_final_publication.py
python scripts/build_taxonomy_palette.py
python scripts/generate_final_domain_taxonomy_figures.py
```

## Reproduce all main and supplementary figures

```bash
bash scripts/final_publication_figures/run_all_portable.sh
```

Every figure is mapped to its input data, responsible script, command, dependencies, random-seed policy and output files in the repository manifests. See `scripts/README.md` and `FIGURE_SCRIPT_REPRODUCIBILITY_MANIFEST.md`.

## Supplementary tables

Supplementary Tables 1–11 are stored under `tables/`. The app reads the same workbooks used by the manuscript and exposes downloadable tables for audit.

## Validation

python -m compileall -q app.py src scripts
